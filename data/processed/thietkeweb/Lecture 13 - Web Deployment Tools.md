## 8.1. Overview of web deployment

**Deploying a website** is the process of transferring website source code from the development environment (**local**) to the production environment (**server**) so that users can access it via the Internet.

### Objectives:

- Ensure the website **operates stably**.
- Make the website **easy to update** and maintain.

# General Steps to Deploy a Website

- **Prepare the source code** files, test them locally. -> Finalize HTML, CSS, and JS
- **Manage the source code** changes in the code. -> Use Git to store and track
- **Choose a hosting platform** VPS, or a private server. -> Examples: Vercel, Netlify,
- **Connect a domain (if available)** for easier user access. -> Configure the domain name
- **Upload or sync the source code to the server** the hosting platform's built-in deployment tools. -> Using Git, FTP, or
- **Test the website after deployment** devices and fix any issues found. -> Test on multiple
