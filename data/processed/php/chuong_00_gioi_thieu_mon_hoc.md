## Tổng Quan Lập Trình Web

### 1. Vai trò của website

Website là kênh quan trọng đối với tổ chức và doanh nghiệp. Website có thể được xem là bộ mặt thứ hai của doanh nghiệp trên Internet, giúp tiếp cận khách hàng, quảng bá sản phẩm và thương hiệu, hỗ trợ hoạt động kinh doanh, bán hàng và nắm bắt xu hướng mua sắm của người tiêu dùng.

Một số con số được nêu trong slide:

- Gần 100% doanh nghiệp hoặc tổ chức có website.
- Ước tính có 1,13 tỷ trang Web.
- Có khoảng 10.500 website được tạo ra mỗi giờ.

### 2. Các loại website

Website có thể được phân loại theo nhiều cách:

| Cách phân loại | Ví dụ |
|---|---|
| Theo quyền sở hữu | Website doanh nghiệp, website cá nhân |
| Theo cấu trúc | Website tĩnh, website động |
| Theo chức năng | Website bán hàng, website tin tức, mạng xã hội, diễn đàn, website giải trí, website rao vặt |
| Theo lĩnh vực hoạt động | Du lịch, xây dựng, giáo dục, thời trang, mỹ phẩm |

## Truy Cập Website Và Mô Hình Client-server

Khi truy cập website, người dùng sử dụng trình duyệt web để gửi yêu cầu đến Web server. Web server xử lý yêu cầu và trả lại nội dung cho trình duyệt. Đây là mô hình Client-server.

### 1. Web server

Web server là chương trình đợi các yêu cầu từ trình duyệt web phía client. Các chức năng chính:

- Cung cấp các trang web.
- Chạy các chương trình điều khiển các cổng giao tiếp như CGI và đưa thông tin qua các cổng đó.
- Điều khiển, kiểm soát các truy cập tới server.

Ví dụ Web server:

- Apache.
- IIS.
- Netscape Web server.

### 2. Các khái niệm cơ bản

Các khái niệm cần nắm:

- Website.
- WebPage.
- WWW.
- Internet.
- Web Browser.
- Web Server.
- Địa chỉ IP.
- Tên miền.
- Port Service.
- URI, chuỗi định vị tài nguyên.

## So Sánh Web Form Và Win Form

| Web form application | Win form application |
|---|---|
| Cài đặt một lần, chạy mọi nơi | Dùng ở đâu, cài ở đó |
| Client không cần cấu hình cao, chỉ cần server có cấu hình tốt | Tất cả máy trạm đều cần cấu hình tốt |
| Dễ bảo trì, nâng cấp | Khó bảo trì, nâng cấp |
| Nghiệp vụ toàn hệ thống nhất quán, dễ kiểm soát | Khi thay đổi nghiệp vụ phải kiểm soát phiên bản phần mềm tương ứng |

## Web Tĩnh Và Web Động

### 1. Web tĩnh

Web tĩnh thường sử dụng HTML, dễ phát triển nhưng tương tác yếu. Người làm web tĩnh thường dùng công cụ trực quan để tạo trang web có nội dung ít thay đổi.

### 2. Web động

Web động khó phát triển hơn, tương tác mạnh hơn và sử dụng nhiều ngôn ngữ khác nhau. Web động thường phải viết nhiều mã lệnh để xử lý nghiệp vụ và dữ liệu.

## Các Ngôn Ngữ Lập Trình Web

Các công nghệ được nhắc đến:

- HTML.
- CSS.
- JavaScript.
- PHP.
- ASP.NET.
- JSP.
- Python.

HTML và CSS dùng để trình bày trang. JavaScript xử lý sự kiện trên client và có thể phối hợp với server. Với các hệ thống có cơ sở dữ liệu phức tạp, cập nhật liên tục hoặc nghiệp vụ phức tạp, các sự kiện thường được chuyển về phía server để xử lý bằng các ngôn ngữ server-side.

Một số nền tảng server-side:

- **ASP.NET:** do Microsoft phát triển trên .NET Framework, công cụ phát triển có thể là Visual Studio.
- **JSP:** Java Server Page, do Sun phát triển, sử dụng Java để xử lý nghiệp vụ tại server.

## Quy Trình Xây Dựng Website

Quy trình xây dựng website gồm các bước chính:

1. Đặc tả yêu cầu.
2. Phân tích và thiết kế.
3. Xây dựng và kiểm thử.
4. Triển khai website.
5. Duy trì, phát triển và quảng bá website.

## Điều Kiện Triển Khai Website

Để triển khai website cần có quyền sử dụng hợp pháp với domain và hosting.

### 1. Domain

Có thể dùng:

- Tên miền con miễn phí, Sub Domain.
- Tên miền riêng.

Khi đăng ký tên miền cần xác định tên tiếng Việt, tên giao dịch tiếng Anh, tên viết tắt và nơi đăng ký. Slide nhấn mạnh nên đăng ký tên miền càng sớm càng tốt.

### 2. Hosting

Có thể:

- Sử dụng hosting miễn phí.
- Thuê không gian riêng.
- Tự host website.

Khi chọn hosting cần xác định môi trường vận hành:

| Máy chủ Windows | Máy chủ Linux |
|---|---|
| Hỗ trợ ASP, PHP, SQL Server, MySQL | Hỗ trợ PHP, JSP, MySQL |
| Đắt hơn Linux | Rẻ hơn Windows |

Ngoài ra cần xác định dung lượng thực tế của website, khả năng mở rộng, băng thông, dịch vụ an toàn, an ninh và backup dữ liệu.
